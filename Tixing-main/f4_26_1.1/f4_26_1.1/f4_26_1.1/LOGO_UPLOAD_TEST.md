# Logo上传功能测试指南

## 修改内容

### APP端 (logo_upload_screen.dart)
1. **批量发送优化**：每5包发送一次，减少ACK等待次数
2. **包大小调整**：从32字节减到16字节，提高BLE传输可靠性
3. **超时处理优化**：连续超时5次才报错，单次超时继续发送
4. **进度显示增强**：显示传输速度和剩余时间

### 硬件端 (logo.c)
1. **批量ACK**：每5包发送一次ACK，与APP对应
2. **幂等处理**：重复包不会导致错误，只返回ACK
3. **序号跳跃检测**：发现丢包时请求重传

### 硬件端 (rx.h)
1. **缓冲区增大**：从512字节增加到1024字节

## 预期性能
- 数据大小：47,432 字节
- 包大小：16 字节
- 总包数：约 2,965 包
- 批量大小：5 包
- 预计时间：60-90 秒

## 测试步骤

1. **重新编译硬件固件**
   - 在 Keil 中重新编译项目
   - 下载到 STM32F4

2. **重新编译APP**
   ```bash
   cd RideWind
   flutter build apk --release
   ```

3. **测试上传**
   - 连接蓝牙设备
   - 进入 Logo 设置界面
   - 选择一张图片
   - 点击"上传到设备"
   - 观察进度条和调试信息

## 调试信息说明
- `共XXX包，预计约XX秒`：开始上传前的预估
- `速度: X.XKB/s，剩余约XX秒`：实时传输状态
- `重传从包X开始`：发生丢包，正在重传
- `✅ 完成！用时XX秒`：上传成功

## 常见问题

### 硬件无响应
- 检查蓝牙连接是否正常
- 检查硬件是否正确接收 LOGO_START 命令
- 查看硬件串口输出

### 重传次数过多
- 蓝牙信号可能不稳定
- 尝试靠近设备
- 检查是否有其他蓝牙干扰

### CRC校验失败
- 数据传输过程中有损坏
- 重新上传
