整体架构
你的 UI 系统基于 240×240 圆形 TFT LCD（SPI2 + DMA），采用状态机驱动，核心由两个函数协作：

Encoder() — 每 20ms 处理旋钮输入和按键事件，驱动 UI 状态切换
LCD() — 每 50ms 刷新屏幕显示
全局变量 ui 控制当前界面编号，chu 控制初始化标志。

三层状态机：

ui3_mode=0: 选灯带模式 (红点)
  旋转 → 切换 Middle/Left/Right/Back (name 0-3)
  单击 → 进入 ui3_mode=1

ui3_mode=1: 选通道模式 (绿点)
  旋转 → 切换 R/G/B (ui3_channel 0-2)，当前通道高亮
  单击 → 进入 ui3_mode=2
  长按 → 退出回 ui3_mode=0

ui3_mode=2: 调节数值模式 (绿点+高亮)
  旋转 → 调节 0-255 (delta×2 加速)，实时更新LED
  单击 → 退出回 ui3_mode=1
关键坐标常量（来自 xuanniu.h）：

灯带指示球：deng_qiu_1_x=50, y=90/130/170
RGB条位置：tiao_1_x=132, y=90/130/170
条宽度：tiao_di_width=127

交互逻辑：

正常模式(红点)：旋钮调节亮度 0-100，大数字实时更新
呼吸灯模式(绿点)：旋钮切换 14 种颜色预设，LED 自动正弦呼吸（3秒周期），颜色条可视化
单击：切换正常/呼吸灯模式
呼吸灯算法：brightness = 0.6 + 0.4 × sin(phase)，20ms 更新一次(50fps)

渲染优化策略
代码中已实现的优化：

脏标记机制：last_num/last_state 缓存，避免重复绘制
局部刷新：只更新变化的数字/颜色条区域，不全屏重绘
DMA 加速：>32字节数据使用 SPI DMA 传输
整数运算：颜色条绘制使用定点数替代浮点
行缓冲区：line_buf[256] 逐行批量写入
