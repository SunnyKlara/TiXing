# 菜单界面重构 - 确认方案

## 📋 文档说明
本文档记录了所有冲突问题的最终解决方案，作为开发的指导文档。

---

## ✅ 已确认的解决方案

### 问题 1：油门模式的按键冲突

**最终方案**：
- 三击进入油门模式后，长按**锁定为加速**（不会触发返回菜单）
- 退出油门模式的方式：
  - 旋转编码器 → 退出到菜单界面
  - 三击编码器 → 退出到菜单界面

**实现要点**：
```c
// 在油门模式下（wuhuaqi_state == 2）
// 1. 禁用长按返回菜单的逻辑
// 2. 长按只执行加速操作
// 3. 旋转或三击时，设置 ui=5（菜单），menu_selected=1（UI1）
```

---

### 问题 2：编码器增量的统一管理

**最终方案**：
在 `Encoder()` 函数开头调用一次 `Encoder_GetDelta()`，保存到全局变量 `encoder_delta`，其他地方使用这个变量。

**实现要点**：
```c
// 在 xuanniu.c 中添加全局变量
int16_t encoder_delta = 0;

void Encoder() {
    // 开头统一读取一次
    encoder_delta = Encoder_GetDelta();
    
    // 各个界面使用 encoder_delta 而不是重复调用
    if (ui == 5) {
        // 菜单界面使用 encoder_delta
        menu_selected += encoder_delta;
    }
    else if (ui == 1) {
        // UI1 使用 encoder_delta
        Num += encoder_delta;
    }
    // ...
}

void LCD() {
    // UI2/UI3/UI4 也使用 encoder_delta
    if (ui == 2 && pei_se_state) {
        deng_num += encoder_delta;
    }
    // ...
}
```

---

### 问题 3：chu 变量的初始化逻辑

**最终方案**：
- 菜单界面使用 **ui=5**（放在最前面，不影响原有逻辑）
- 菜单对应 **chu=5**（初始化）和 **chu=0**（已初始化）
- 保持原有 UI1-4 的 chu 逻辑不变

**chu 变量状态表**：
```
chu=5: 需要初始化菜单界面
chu=0: 菜单已初始化
chu=1: 需要初始化 UI1
chu=2: 需要初始化 UI2
chu=3: 需要初始化 UI3
chu=4: 需要初始化 UI4
```

**界面切换逻辑**：
```c
// 开机
ui=5, chu=5

// 从菜单进入 UI1
ui=5 → ui=1, chu=0 → chu=1

// 从 UI1 返回菜单
ui=1 → ui=5, chu=1 → chu=5

// 从菜单进入 UI2
ui=5 → ui=2, chu=0 → chu=2

// 从 UI4 返回菜单（修改原有逻辑）
ui=4 → ui=5, chu=4 → chu=5  // 原来是 chu=4 → chu=1
```

---

### 问题 4：菜单界面的默认选中

**最终方案**：
- 开机默认选中 **UI1**（menu_selected = 1）
- 返回菜单时选中**上次使用的 UI**（记忆功能）

**实现要点**：
```c
// 全局变量
uint8_t menu_selected = 1;  // 当前选中的 UI（1-4）

// 从 UI3 返回菜单
ui = 5;
chu = 5;
menu_selected = 3;  // 记住是从 UI3 返回的

// 用户旋转编码器
menu_selected += encoder_delta;
if (menu_selected > 4) menu_selected = 1;
if (menu_selected < 1) menu_selected = 4;
```

### 问题 5：菜单界面的"逃生出口"

**最终方案**：
不需要特殊处理，用户总是会选择进入某个 UI。

**说明**：
- 菜单界面的长按不执行任何操作（已经在菜单中）
- 用户只能通过单击进入某个 UI

---

### 问题 6：蓝牙命令的语义

**最终方案**：
APP 发送 `UI:X` 命令时，先切换到菜单，选中对应 UI，然后自动进入（模拟用户操作）。

**实现要点**：
```c
// 接收到 "UI:2\n"
ui = 5;           // 先切换到菜单
chu = 5;          // 需要初始化菜单
menu_selected = 2; // 选中 UI2
auto_enter = 1;   // 标志位：自动进入

// 在 LCD() 中
if (ui == 5 && auto_enter) {
    // 等待菜单初始化完成后自动进入
    ui = menu_selected;
    chu = menu_selected;
    auto_enter = 0;
}
```

### 问题 7：UI2/UI3/UI4 的子状态保持

**最终方案**：
每次进入都**重置为初始状态**。

**实现要点**：
```c
// 在 LCD() 的初始化逻辑中
if (ui == 2 && chu == 2) {
    // 初始化 UI2
    pei_se_state = 0;  // 重置子状态
    chu = 3;
}

if (ui == 3 && chu == 3) {
    // 初始化 UI3
    tiao_se_num = 0;   // 重置子状态
    chu = 4;
}

if (ui == 4 && chu == 4) {
    // 初始化 UI4
    brt_state = 0;     // 重置子状态
    chu = 5;           // 修改：原来是 chu=1，现在改为 chu=5
}
```

### 问题 9：菜单界面的刷新策略

**最终方案**：
借鉴现有的界面切换方法，使用边框高亮（不重绘图标，只绘制边框）。

**实现要点**：
```c
// 只在选中项变化时重绘
static uint8_t last_selected = 0;

void Draw_Menu() {
    if (menu_selected != last_selected) {
        // 清除旧边框
        Draw_Border(last_selected, 0);
        // 绘制新边框
        Draw_Border(menu_selected, 1);
        last_selected = menu_selected;
    }
}
```

### 问题 10：蓝牙命令的扩展

**最终方案**：
暂时不考虑 APP 端的菜单控制，只需要确保界面切换时状态同步。

**说明**：

- 不新增 `MENU\n` 命令
- 硬件端的菜单操作不需要通过蓝牙通知 APP

## 🎯 核心实现要点总结

### 1. 新增全局变量

```c
// 在 xuanniu.h 中声明
extern uint8_t menu_selected;  // 菜单选中的 UI（1-4）
extern int16_t encoder_delta;  // 编码器增量（统一读取）
extern uint8_t auto_enter;     // 蓝牙命令自动进入标志

// 在 xuanniu.c 中定义
uint8_t menu_selected = 1;
int16_t encoder_delta = 0;
uint8_t auto_enter = 0;
```

### 2. 修改 chu 变量逻辑

```c
// 开机初始化
ui = 5;
chu = 5;
menu_selected = 1;

// UI4 返回菜单时
if (ui == 4 && 长按) {
    ui = 5;
    chu = 5;  // 修改：原来是 chu = 1
}
```

### 3. 油门模式的特殊处理

```c
// 在 Encoder() 中
if (wuhuaqi_state == 2) {
    // 油门模式下，禁用长按返回菜单
    if (长按) {
        // 执行加速逻辑
    }
    if (旋转 || 三击) {
        // 退出油门模式
        wuhuaqi_state = 0;
        ui = 5;
        chu = 5;
        menu_selected = 1;
    }
}
else {
    // 非油门模式下，长按返回菜单
    if (长按 && ui != 5) {
        menu_selected = ui;  // 记住当前 UI
        ui = 5;
        chu = 5;
    }
}
```

### 4. 菜单界面的绘制逻辑

```c
// 在 LCD() 中
if (ui == 5) {
    if (chu == 5) {
        // 初始化菜单
        LCD_Fill(0, 0, LCD_W, LCD_H, BLACK);
        Draw_Menu_Items();
        chu = 0;
    }
    
    // 更新选中项
    Update_Menu_Selection();
    
    // 处理单击进入
    if (单击) {
        ui = menu_selected;
        chu = menu_selected;
    }
    
    // 处理蓝牙自动进入
    if (auto_enter) {
        ui = menu_selected;
        chu = menu_selected;
        auto_enter = 0;
    }
}
```

## ⚠️ 需要注意的细节

1. **chu 变量的循环**：
   - 原来：chu=4 → chu=1（UI4 返回 UI1）
   - 现在：chu=4 → chu=5（UI4 返回菜单）

2. **编码器增量的消费**：
   - 确保 `Encoder_GetDelta()` 只在 `Encoder()` 开头调用一次
   - 其他地方使用 `encoder_delta` 变量

3. **油门模式的判断**：
   - 在处理长按返回菜单前，先判断 `wuhuaqi_state != 2`

4. **菜单选中项的边界**：
   - `menu_selected` 的范围是 1-4
   - 旋转时需要循环：1→2→3→4→1

5. **子状态的重置时机**：
   - 在 `chu == X` 的初始化逻辑中重置
   - 不在返回菜单时重置（避免重复操作）
